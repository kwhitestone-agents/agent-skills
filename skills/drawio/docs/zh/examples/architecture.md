# 架构图示例

创建架构图的示例提示词。

## 三层架构

> "绘制一个三层 Web 应用架构：
> - 客户端层：Web 浏览器和移动应用
> - 应用层：负载均衡器、Web 服务器、API 服务器
> - 数据层：主数据库、只读副本、缓存"

## 微服务架构

> "创建一个微服务架构，包含：
> - 顶部的 API 网关
> - 认证服务、用户服务、订单服务、支付服务
> - 每个服务都有自己的数据库
> - 所有服务通过 API 网关连接"

## CI/CD 流水线

> "绘制 CI/CD 流水线架构：
> - 开发者提交代码
> - Git 仓库
> - 构建服务器编译代码
> - 运行自动化测试
> - 如果测试通过：部署到预发布环境
> - 手动批准
> - 部署到生产环境"

## 云基础设施

> "创建云基础设施数据：
> - 互联网 → 负载均衡器
> - Web 服务器（自动扩缩容组）
> - 应用服务器（自动扩缩容组）
> - 数据库（主库 + 备库）
> - 用于静态内容的 CDN"

## 事件驱动架构

> "绘制事件驱动架构：
> - 事件生产者：Web API、定时任务
> - 消息代理（Kafka/RabbitMQ）
> - 事件消费者：邮件服务、分析服务、通知服务
> - 数据仓库"

## Kubernetes 部署

> "创建 Kubernetes 部署架构：
> - Ingress 控制器
> - 服务：前端、后端、数据库
> - Pod：每个服务的多个实例
> - 用于数据的持久卷
> - ConfigMaps 和 Secrets"

## 架构图技巧

- **使用分层结构**：清楚地描述层或级
- **显示数据流**：指示数据如何在组件之间移动
- **标记连接**：描述使用什么协议或 API
- **分组相关组件**：使用容器或边界
