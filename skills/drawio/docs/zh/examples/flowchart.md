# 流程图示例

使用 Draw.io 技能创建流程图的示例提示词。

## 简单流程

> "创建一个包含 4 个步骤的简单流程：开始 → 初始化 → 处理 → 结束"

## 决策流程

> "绘制用户登录流程图：
> - 用户输入凭据
> - 检查凭据是否有效
> - 如果有效：登录成功
> - 如果无效：显示错误并重试"

## 循环流程

> "创建一个数据处理循环流程图：
> - 开始
> - 读取数据
> - 还有更多数据吗？（决策）
> - 如果是：处理数据 → 返回读取
> - 如果否：保存结果 → 结束"

## 复杂业务流程

> "绘制订单处理流程图：
> - 客户下单
> - 验证库存
> - 如果缺货：通知客户 → 结束
> - 如果有货：处理支付
> - 如果支付失败：重试支付（最多 3 次）
> - 如果支付成功：发货
> - 发送确认 → 结束"

## 错误处理流程

> "创建一个包含错误处理的流程图：
> - 开始处理
> - 尝试执行操作
> - 如果发生错误：记录错误
> - 检查错误是否可恢复
> - 如果是：重试操作
> - 如果否：警告管理员 → 结束
> - 如果没有错误：继续下一步 → 结束"

## 并行处理

> "绘制一个显示并行任务的流程图：
> - 开始
> - 分为 3 个并行任务：任务 A、任务 B、任务 C
> - 等待所有任务完成
> - 合并结果
> - 结束"

## 流程图提示词技巧

- **明确连接**：使用 "→" 或 "然后" 显示流程
- **描述决策点**：使用 "如果/则/否则" 语言
- **指定节点类型**：提及 "开始"、"结束"、"决策"、"处理"
- **包含循环**：清楚地描述何时返回到之前的步骤

## 自己试试

将任何这些提示词复制到 Claude Code 中，查看结果！
