# 创建图表

学习如何用自然语言创建各种类型的图表。

## 支持的图表类型

### 流程图

描述流程步骤和决策点：

> "创建一个电商结账流程图：
> - 用户添加商品到购物车
> - 用户进入结账流程
> - 用户输入配送信息
> - 用户选择支付方式
> - 处理支付
> - 订单确认或支付失败"

### 架构图

描述系统组件及其关系：

> "绘制一个三层 Web 应用架构：
> - 客户端层：Web 浏览器和移动应用
> - 应用层：负载均衡器、Web 服务器、API 服务器
> - 数据层：主数据库、只读副本、缓存"

### 序列图

描述组件之间随时间的交互：

> "创建一个用户认证序列图：
> - 用户 -> 登录页面：输入凭据
> - 登录页面 -> 认证服务：验证凭据
> - 认证服务 -> 数据库：查询用户
> - 数据库 -> 认证服务：返回用户数据
> - 认证服务 -> 登录页面：返回认证令牌
> - 登录页面 -> 用户：重定向到仪表板"

## 最佳实践

### 明确具体

提供清晰、详细的描述：

**好的示例**：
> "创建一个包含 5 个步骤的流程图：开始、验证输入、处理数据、保存结果、结束"

**效果不佳**：
> "画一个流程图"

### 使用结构化列表

将复杂图表分解为步骤或组件：

> "创建一个微服务架构，包含：
> - API 网关
> - 认证服务
> - 用户服务
> - 订单服务
> - 支付服务
> - 通知服务
> - 每个服务都连接到 API 网关"

### 指定视觉属性

你可以请求特定的样式：

> "创建一个流程图，其中：
> - 开始/结束节点使用圆角矩形，绿色填充
> - 处理节点使用矩形，蓝色填充
> - 决策节点使用菱形，黄色填充"

## 获得更好效果的技巧

1. **从简单开始**：先创建基本图表，然后增加复杂度
2. **迭代优化**：通过多个提示词优化图表
3. **使用示例**：参考现有图表模式
4. **明确表达**：清楚地说明节点数量和关系
